# Historias de Usuario - HidroSmart Mobile

## Propósito
Este documento reúne las historias de usuario principales para el proyecto móvil de HidroSmart, con base en la funcionalidad actual y los requisitos identificados.

## Historias de Usuario

### HU-00: Estructura base del proyecto y arquitectura en capas
Como desarrollador del proyecto, quiero contar con una estructura base del proyecto organizada en capas (presentación, aplicación, dominio, infraestructura y datos), para facilitar el mantenimiento, la escalabilidad y la evolución del sistema.

### HU-01: Inicio de sesión
Como usuario, quiero ingresar a la aplicación con mis credenciales, para acceder a mis hogares y dispositivos de forma segura.

### HU-02: Registro de usuario
Como nuevo usuario, quiero registrarme en la plataforma, para crear mi cuenta y comenzar a usar HidroSmart.

### HU-03: Autenticación de segundo factor
Como usuario registrado, quiero verificar mi identidad con 2FA, para proteger mi cuenta y mis datos.

### HU-04: Visualización del panel principal
Como usuario, quiero ver el tablero principal de HidroSmart dentro de la aplicación móvil, para consultar el estado general de mi cuenta.

### HU-05: Consulta de hogares y dispositivos
Como usuario, quiero ver mis hogares y dispositivos asociados, para conocer su estado en cualquier momento.

### HU-06: Monitoreo de consumo
Como usuario, quiero consultar el consumo de agua o energía de mis dispositivos, para monitorear su uso.

### HU-07: Gestión de dispositivos
Como usuario, quiero registrar y administrar mis dispositivos, para mantener actualizada la información de mi instalación.

### HU-08: Gestión de hogares
Como usuario, quiero agregar o editar hogares, para organizar mis instalaciones según mis necesidades.

### HU-09: Privacidad y consentimiento
Como usuario, quiero revisar y aceptar las políticas de privacidad, para conocer cómo se tratarán mis datos.

### HU-10: Experiencia móvil fluida
Como usuario, quiero que la aplicación se abra correctamente en móvil, con carga y navegación fluidas, para usarla sin interrupciones.

### HU-11: Restricción de acciones de creación
Como usuario, quiero que la aplicación impida acciones de creación innecesarias en la vista web embebida, para evitar cambios no deseados desde la versión móvil.

### HU-12: Compatibilidad multiplataforma
Como usuario, quiero que la aplicación funcione correctamente en Android, iOS y web, para usarla desde cualquier dispositivo.
